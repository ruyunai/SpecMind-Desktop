# 产品背景

## 存在原因
传统 ToB 项目 PRD 产出存在三大痛点：
1. **撰写慢**：销售输入杂乱（微信记录、口头承诺、非标准文档），PM 手动整理耗时，平均 4 小时/份
2. **评审反复打回**：PRD 质量参差，合规/成本/技术可行性未前置校验，评审多轮返工
3. **落地时需求变更**：合同条款与 PRD 脱节，过度承诺未识别，交付阶段频繁变更

## 目标用户
| 角色 | 使用场景 |
|------|---------|
| ToB 企业 PM | 日常产出标准化 PRD，提升一次通过率 |
| 销售负责人 | 导入脏需求，获取清洗报告与报价方案 |
| 法务/合规岗 | 基于 Legal 预检结果做正式复核（辅助而非替代） |
| 技术负责人 | 评审 PRD 技术可行性 |
| 项目交付经理 | 基于 Planner 输出交付计划 |
