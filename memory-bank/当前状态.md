## 当前架构概览

```
GUI (PySide6, 三栏深色主题)
 ├─ 左：资产库（ChromaDB 企业文档，支持上传）
 ├─ 中：工作区（需求输入 → PRD → 附件导出）
 └─ 右：工作流画布（7 节点状态 + 拓扑图）

Orchestrator (LangGraph + QThread)
 ├─ SAR Agent   → RAG(标准功能库) → LLM → 需求清洗
 ├─ Legal Agent → RAG(法规库)     → LLM → 合规预检
 ├─ PM Agent    → LLM → PRD 生成
 ├─ Commercial  → 公式计算 → 双报价（不调用 LLM）
 ├─ Contract    → RAG(合同模板库) → LLM → 条款比对
 ├─ Review      → LLM → 三维评审
 └─ Planner     → LLM → 交付计划

存储层
 ├─ ChromaDB   → 向量检索（企业上传的文档）
 ├─ SQLite     → FTS5 BM25 + 审计日志 + 资产 FTS
 └─ 文件系统    → 日志 (JSON Lines + RotatingFileHandler)

安全
 ├─ 密钥加密   → Fernet + PBKDF2 + 机器绑定
 └─ API Key   → secrets.enc 加密存储
```

## 已知问题

1. **检索相似度偏低**（非阻塞）：上传文档少时，RAG 相似度 <0.6 触发降级提示。这是正常设计——企业文档越多，检索越精准。
2. **FTS5 LIKE 回退噪音**（性能无影响）：中文分词无 jieba 依赖，trigram 主路径召回率足够，LIKE 回退仅用于 <3 字符短 token，日志 WARNING 可忽略。
3. **stream_mode 信号滞后**（P2 架构限制）：LangGraph 0.2.x updates 模式在节点返回后才 yield，GUI "执行中"状态短暂。
4. **exe 未代码签名**：SmartScreen 会拦截，发布给用户前需要签名。
